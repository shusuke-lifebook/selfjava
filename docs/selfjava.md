## 📒 2. Javaの基本

### 📒 2.1. 変数

- **変数**とは一言でいうと「データの入れ物」

#### 📒 2.1.1. 変数の宣言

- **変数**を利用するにはまず、変数を**宣言**する。

  ```Java
  データ型 変数名 [= 初期値][,...]
  ```

#### 📒 2.1.2. 識別子の命令規則

- Javaでは次のルールに則って識別子を命名する。
  1. すべてのUnicode文字を利用できる
  2. ただし、1文字目は数字以外であること
  3. アルファベットの大文字／小文字は区別される
  4. 予約語でないこと
  5. 文字数の制限はない

#### 📒 2.1.3. よりよい識別子のためのルール

- 命名規則ではないが、コード読みやすくするという意味で以下の点を気をかけておきたい。
  1. 名前からデータの内容が類推できる。
  2. 長すぎない、短すぎない
  3. ローマ字での命名は避ける
  4. 見た目にまぎらわしくない
  5. 1文字目のアンダースコアは、特別な意図を想定するので避ける
  6. 記法を統一する

#### 📒 2.1.4. 定数

- **定数の利用** 定数を宣言するには、finalという気ワードを利用する
  ```Java
  final データ型 定数名 = 値
  ```

### 📒 2.2. データ型

- **データ型(型)**とは、プログラムの中で扱うデータの種類のこと。

#### 📒 2.2.1. データ型の分類

- データ型は、**基本型**と**参照型**に分類される
  1. 基本型：値そのものが格納される。
  2. 参照型：値の格納場所を表す情報(メモリ上のアドレスのようなもの)を格納する。

#### 📒 2.2.2. 基本型の種類

- 基本型は**プリミティブ(原始)型**と呼ばれることがある。

#### 📒 2.2.3. 整数型

- 整数型は可能できる値の範囲によって、以下の5種類に分類できる
  - byte
  - short
  - int
  - long
  - char

#### 📒 2.2.4 浮動小数点型

- 浮動小数点型は、小数点を表すデータ型です。
  - float
  - double
- 著しくリソースが制限されるなど、メモリへの負担が懸念されるような状況でなければ、値範囲も大きく精度にも優れたdoubleを優先して利用する。
